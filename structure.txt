2.  RELATIONSHIPS  (source -[REL]-> target)
────────────────────────────────────────────────────────────────────────────────

  2a.  Paper — authorship & structure
  ─────────────────────────────────────

  Paper -[AUTHORED_BY]-> Author
    Who wrote the paper.
    Properties : none

  Author -[CO_AUTHORED]-> Author  
    Two authors who share at least one paper together.
    Properties :
      paper_count   int     number of co-authored papers (incremented on MERGE)

  Paper -[HAS_SECTION]-> Section
    Ordered sections extracted from the PDF.
    Properties : none

  Section -[NEXT_SECTION]-> Section  
    Linked-list ordering of sections within the same paper.
    Enables walking context windows during RAG retrieval.
    Properties : none


  2b.  Paper — knowledge extraction
  ────────────────────────────────────

  Paper -[INTRODUCES]-> Concept
    Paper coins or defines this concept for the first time.
    Properties :
      source        string    "llm" | "pdf"
      confidence    float     0.0 – 1.0  (LLM-assigned)  

  Paper -[APPLIES]-> Concept
    Paper uses an existing concept without introducing it.
    Properties :
      source        string    "llm" | "pdf"
      confidence    float     0.0 – 1.0  

  Paper -[USES_METHOD]-> Method
    Method employed in the paper.
    Properties :
      source        string    "llm" | "pdf"
      confidence    float     0.0 – 1.0  

  Paper -[USES_DATASET]-> Dataset
    Dataset used for training or evaluation.
    Properties :
      source        string    "llm" | "pdf"
      confidence    float     0.0 – 1.0  

  Paper -[BELONGS_TO]-> Topic
    Paper belongs to a research area.
    Properties :
      source        string    "llm"
      confidence    float     0.0 – 1.0  


  2c.  Paper — inter-paper
  ──────────────────────────

  Paper -[CITES]-> Paper
    Direct citation extracted from PDF bibliography.
    Target may be a stub node (stub: true) if not yet ingested.
    Properties : none

  Paper -[EXTENDS]-> Paper
    Paper directly builds on or extends prior work.
    Properties :
      source        string    "llm"
      confidence    float     0.0 – 1.0  

  Paper -[CONTRADICTS]-> Paper
    Paper disputes findings of another paper.
    Properties :
      source        string    "llm"
      confidence    float     0.0 – 1.0  


  2d.  Knowledge — inter-node
  ─────────────────────────────

  Concept -[RELATED_TO]-> Concept
    Conceptually linked ideas extracted by LLM.
    Properties :
      source        string    "llm"
      confidence    float     0.0 – 1.0  

  Method -[EXTENDS]-> Method  
    One method builds on another  (e.g. LoRA -> PEFT -> Transformer).
    Properties :
      source        string    "llm"
      confidence    float     0.0 – 1.0

  Topic -[BELONGS_TO]-> Topic
    Sub-topic hierarchy  (e.g. GAN -> Deep Learning -> Machine Learning).
    Properties : none

3.  CONSTRAINTS & INDEXES
────────────────────────────────────────────────────────────────────────────────

  UNIQUE CONSTRAINTS
  ──────────────────
  CREATE CONSTRAINT paper_arxiv_id   IF NOT EXISTS FOR (p:Paper)   REQUIRE p.arxiv_id    IS UNIQUE
  CREATE CONSTRAINT author_id        IF NOT EXISTS FOR (a:Author)  REQUIRE a.author_id   IS UNIQUE
  CREATE CONSTRAINT section_id       IF NOT EXISTS FOR (s:Section) REQUIRE s.section_id  IS UNIQUE
  CREATE CONSTRAINT concept_name     IF NOT EXISTS FOR (c:Concept) REQUIRE c.name_norm   IS UNIQUE
  CREATE CONSTRAINT method_name      IF NOT EXISTS FOR (m:Method)  REQUIRE m.name_norm   IS UNIQUE
  CREATE CONSTRAINT dataset_name     IF NOT EXISTS FOR (d:Dataset) REQUIRE d.name_norm   IS UNIQUE
  CREATE CONSTRAINT topic_name       IF NOT EXISTS FOR (t:Topic)   REQUIRE t.name_norm   IS UNIQUE

  VECTOR INDEX  (for RAG retrieval)
  ──────────────────────────────────
  CREATE VECTOR INDEX section_embedding IF NOT EXISTS
    FOR (s:Section) ON s.embedding
    OPTIONS { indexConfig: { `vector.dimensions`: 768, `vector.similarity_function`: 'cosine' } }

  REGULAR INDEXES  (query performance)
  ──────────────────────────────────────
  CREATE INDEX paper_year    IF NOT EXISTS FOR (p:Paper)  ON (p.year)
  CREATE INDEX paper_stub    IF NOT EXISTS FOR (p:Paper)  ON (p.stub)
  CREATE INDEX paper_cat     IF NOT EXISTS FOR (p:Paper)  ON (p.category)
  CREATE INDEX author_orcid  IF NOT EXISTS FOR (a:Author) ON (a.orcid)

────────────────────────────────────────────────────────────────────────────────
5.  QUICK DESIGN NOTES
────────────────────────────────────────────────────────────────────────────────

  - Embeddings live only on Section nodes (RAG backbone).
    Paper has no embedding — retrieval is at section granularity.

  - INTRODUCES vs APPLIES distinguishes novel vs applied concept usage.

  - stub: true on Paper marks citation-only stubs awaiting ingestion.
    Query uningested stubs:
      MATCH (p:Paper {stub: true}) RETURN p.arxiv_id ORDER BY p.arxiv_id

  - confidence float on LLM edges enables quality-filtered queries:
      MATCH (p:Paper)-[r:INTRODUCES {source:'llm'}]->(c:Concept)
      WHERE r.confidence >= 0.8 RETURN p.title, c.name

  - Topic hierarchy enables drill-down:
      MATCH (t:Topic)-[:BELONGS_TO*]->(root:Topic {name_norm:'machine learning'})
      RETURN t.name

  - Method hierarchy mirrors Topic hierarchy:
      MATCH (m:Method)-[:EXTENDS*]->(root:Method {name_norm:'transformer'})
      RETURN m.name

  - NEXT_SECTION enables context-window expansion for RAG:
      MATCH (s:Section {section_id: $hit_id})-[:NEXT_SECTION]->(next:Section)
      RETURN s.content + next.content AS context

  - CO_AUTHORED enables collaboration network queries:
      MATCH (a:Author {name_norm: 'vaswani'})-[:CO_AUTHORED*1..2]-(collab:Author)
      RETURN DISTINCT collab.name
